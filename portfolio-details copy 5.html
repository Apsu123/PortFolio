Othello Game – C++ / Qt Desktop Application

I built a desktop version of Othello (Reversi) using C++ and Qt. Getting the full game logic to work correctly was a rewarding challenge — Qt’s event-driven architecture can be tricky to navigate at first. The project taught me a lot about low-level programming concepts and how to efficiently manage state and UI updates in real time.

The game engine handles complex move validation and flipping logic smoothly, showcasing the performance and precision of C++. While the board visuals turned out nicely, I focused more on making the gameplay feel responsive and bug-free than on fancy UI design.

I had initially planned to include an AI opponent using minimax with heuristics, but due to time constraints, it didn’t make it into the final version. Even so, the project solidified my understanding of game loops, state management, and GUI event handling in Qt.

Looking back, I’d refactor portions of the logic for clarity and scalability — but overall, I’m proud of how well the core mechanics and logic hold up.
